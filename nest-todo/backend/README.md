

API Endpoints and Usage
Base URL:
bash

http://localhost:3000/api
Endpoints:
GET /todos:

Fetch all todos or filter by query parameters.
Parameters:
status: Filter by todo status (e.g., 'OPEN', 'INPROGRESS', 'COMPLETE').
priority: Filter by todo priority (e.g., 'HIGH', 'MEDIUM', 'LOW').
todays todos: filter to get todos of the current day
POST /todos:

Create a new todo.
Request Body:
json
{
  "title": "New Todo",
  "description": "Description of the new todo"
}
PATCH /todos/:id:

Update todo status and priority.
Request Body:
json

{
  "status": "INPROGRESS",
  "priority": "HIGH"
}
DELETE /todos/:id:

Delete a todo by ID.
Database Schema Description
The application uses a relational database with the following schema:

Users Table:

id: Integer (Primary Key)
email: String (Unique)
password: String (Hashed)
salt: String
--------------------
Todos Table:

id: Integer (Primary Key)
title: String
description: String
status: String ('OPEN', 'INPROGRESS', 'COMPLETE')
priority: String ('HIGH', 'MEDIUM', 'LOW')
userId: Integer (Foreign Key referencing Users Table)
------------------------------
Third-Party Libraries and Tools Used
NestJS: Backend framework for building scalable and maintainable server-side applications.
React: Frontend library for building user interfaces.
Axios: HTTP client for making API requests.
Bcrypt.js: Library for hashing and salting passwords.
JWT (JSON Web Tokens): Used for authentication and authorization.
How to run:
Change the typeorm configuration to your maria db connection and create a todomanagement db in order to store the information
run npm install to install node module and dependencies 
run the app using npm run start:dev
